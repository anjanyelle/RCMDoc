Module 5: Patient Check-in - Flow Documentation

Version: 1.0

Module ID: MOD-005

Category: Patient Management

1. Module Overview

Purpose: Check-in patients when they arrive for appointments, verify information, collect copay.

Why Hospitals Use It: Streamline patient flow, update information, collect payments upfront.

Main Users: Front Desk Staff, Patients (self-check-in kiosk)

2. Actors Involved



┌──────────────────────────────────────────────────────┐
│ ACTORS IN PATIENT CHECK-IN MODULE                    │
├──────────────────────────────────────────────────────┤
│                                                      │
│ 1. Front Desk Staff                                  │
│    - Checks in patients                              │
│    - Verifies demographics                           │
│    - Updates insurance information                   │
│    - Collects copay                                  │
│    - Verifies appointment                            │
│                                                      │
│ 2. Patient                                           │
│    - Provides updated information                    │
│    - Shows insurance card / ID proof                │
│    - Signs consent forms                             │
│    - Completes intake forms                          │
│    - Makes copay payment                             │
│                                                      │
│ 3. Doctor / Provider                                 │
│    - Gets patient arrival notification               │
│    - Views patient status (Checked-In)               │
│    - Starts consultation                             │
│                                                      │
│ 4. Insurance Verification Team/System                │
│    - Re-check insurance eligibility                  │
│    - Verify copay / deductible                       │
│    - Validate coverage                               │
│                                                      │
│ 5. Billing Team (Optional)                           │
│    - Handles outstanding balances                    │
│    - Verifies pending patient dues                   │
│                                                      │
│ 6. System                                            │
│    - Check-in Engine                                 │
│    - Queue Management                                │
│    - Payment Processor                               │
│    - Appointment Validation Engine                   │
│    - Consent Management                              │
│    - Notification Engine                             │
│                                                      │
│ 7. External APIs                                     │
│    - Stripe (Payment processing)                     │
│    - Twilio (SMS/Doctor notification)                │
│    - DocuSign (E-signature)                          │
│    - Availity API (Insurance eligibility)            │
│    - EMR/EHR Integration API                         │
│                                                      │
│ 8. Self Check-In Kiosk / Patient Portal (Optional)   │
│    - Self check-in                                   │
│    - Update details                                  │
│    - Digital form submission                         │
│    - Online payment                                  │

10. Error Scenarios


Error 1: Payment Failed
   ↓
Show error message
   ↓
Options:
1. Retry payment
2. Try different card
3. Pay later (mark as pending)
Error 2: No Appointment Found
   ↓
Show error
   ↓
Options:
1. Search manually
2. Walk-in registration
Error 3: Already Checked In
   ↓
Show message
   ↓
Display queue number
